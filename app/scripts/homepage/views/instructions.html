Instructions

Angular allows you to use animations in your projet once you include the angular-animate dependency.

There are two ways to make use of animations when ngAnimate is used:

  • CSS
  • JavaScript.

Once you include ngAnimate in your directive, Angular will start to interpret directives (ng-repeat, ng-if, ng-show, etc) whose behaviour introduces a change in the DOM.

For these directives, Angular will add CSS classes which looks like:

  • .ng-{EVENT}
  • .ng-{EVENT}-active
ng-show and ng-hide are the sole exceptions for whom Angular will not add the classes .ng-{EVENT} or .ng-{EVENT}-active, but will instead add the class: .ng-hide.

As a consequence, if one wants to add animation, he needs to override the corresponding classes added by Angular.

To sum up, this table indicates the classes to use for each directive:

Angular directive Event Class
ng-repeat enter .ng-enter, .ng-enter-active
leave .ng-leave, .ng-enter-leave
move .ng-move, .ng-enter-move
ng-view, ng-include, ng-switch, ng-if, ng-message enter .ng-enter, .ng-enter-active
leave .ng-leave, .ng-enter-leave
ng-class add .ng-add, .ng-add-active
remove .ng-remove, .ng-enter-remove
ng-show, ng-hide add add class .ng-hide
remove remove class .ng-hide
form, ng-model valid ng-valid
invalid ng-invalid
[...] any other validations that are performed within the form or on the model itself
ng-messages active .ng-active
inactive .ng-inactive
